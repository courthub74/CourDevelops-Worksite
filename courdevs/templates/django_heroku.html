Build Deployment To


  1. Login to Heroku
  2. in the (cmdline) type: heroku login
    you will be brought to a login page just accept

  3. Create the app
  4. in the (cmdline) type: heroku create
    you have the option to name your app here (heroku create appname)

  5. Secret Key Settings
  6. in settings.py:

    copy the SECRET_KEY = '0984309240943skfiekfDILS###' <--just an example

    go to your Heroku project and click settings

    click Reveal Config Vars

    paste the key(i.e. '0984309240943skfiekfDILS###') where it says 'VALUE'

    paste 'SECRET_KEY' where it says 'KEY'

  7. Installations
  8. Go back to terminal and install:

    pip install gunicorn

    pip install django-heroku

    pip install python-decouple

    NOTE: Do a pip freeze to check if its all there

  9. Imports
  10. in settings.py underneath import os type these:

    import django_heroku

    from decouple import config

    import dj_database_url


  11. Local Settings
  12. at bottom of settings.py under everything type:

    django_heroku.settings(locals())


  13. For Secret Key in Settings
  14. in settings.py:

    type in: SECRET_KEY = config('SECRET_KEY')

    in (cmdline) touch .env [this creates an .env file]

    cut and paste the original SECRET_KEY into the .env file save and close

    in (cmdline) touch .gitignore [this creates a .gitignore file]

    type .env in there save and close

  15. Middleware
  16. go to settings.py under MIDDLEWARE:

    underneath 'django.middleware.security.SecurityMiddleware', type:

    'whitenoise.middleware.WhiteNoiseMiddleware'

  17. Static Files
  18. in settings.py under STATICFILES_DIRS = [ os.path.join(BASE_DIR, 'static') ] type:

    STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'

  19. Procfile
  20. in (cmdline)touch Procfile [this creates a Procfile] <--Case Sensitive

    in Procfile, type:

    web: gunicorn (name of directory with wsgi file in it).wsgi

    save and close Procfile

  21. requirements.txt
  22. in (cmdline) type pip freeze > requirements.txt [this creates the requirements.txt file]

  23. Static Directory for 'Collectstatic'
  24. If you haven't already for CSS, JS and Images purposes, make sure you add a directory called static in the main project folder

    If nothing is in it just place a blank file in the static folder and call it __init__.py

    This is for the purpose of the 'CollectStatic' when deploying

  25. Push To Heroku
  26. in (cmdline) type:

    git push heroku master

    before you do this, make sure you commit to git

  27. Heroku Migrates
  28. in (cmdline) type:

    heroku run python manage.py makemigrations

    heroku run python manage.py migrate

    heroku run python manage.py createsuperuser
    set up name | email | password

    heroku run python manage.py collectstatic
