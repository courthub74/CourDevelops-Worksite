Setting up


  1. Initial Setup and Installation
  2. mkdir (project)
    inside above folder, install virtualenv (virtualenv 'venv')
    turn on virtualenv (. venv/Scripts/activate)
    while in virtualenv, install django (pip install django)
    create a project (django-admin.py startproject 'project name')

    NOTE: if not working try without .py

    open text editor and attach the folder to it
    in your (project) folder runserver: python manage.py runserver
    then in same folder: python manage.py migrate (creates admin screen)
    then in same folder: python manage.py createsuperuser
    NOTE: if doesn't work use:
    winpty python manage.py createsuperuser
    set up user name and password


  3. Starting the App

  4. then in same folder: python manage.py startapp (newapp)

    *(newapp) is the name of the newly created app included are the admin | apps | models | tests | urls | views

    in your new app folder create a 'urls.py' file

    in main folder(project) SETTINGS.py add the newly created app (newapp) under 'INSTALLED APPS'


    in (newapp) urls.py add the bear bones:

    from django.urls import path
    urlpatterns = [

    ]



    create a VIEW or VIEWS:

    the views.py is created automatically in your (newapp)
    to create a view is similar to creating a new page in
    flask, with a function. so for basic home page you would do:

    def home(request): <--pass a request thru function because the browser is requesting that a page be shown
    return render(request, "home.html", {})

    FOR EACH PAGE ON YOUR SITE YOU CREATE ONE OF THESE!
    about me | about them | next page | etc.


    create TEMPLATES to match each VIEW:

    inside the (newapp) folder, create a 'templates' folder for html files

    and layout the desired html files that you created in VIEWS



    go to (newapp) urls.py, import all from views and add the paths for each view:

    example:

    from django.urls import path from . import views

    urlpatterns = [
    path(' ', views.home, name='home'),
    path('about/', views.about, name='about'),
    ]


    NOW go to (project) urls.py, and INCLUDE the (newapp) urls:

    exemple:

    from django.contrib import admin
    from django.urls import path, include

    urlpatterns = [
    path('admin/', admin.site.urls),
    path(' ', include('newapp.urls')),
    ]


    NOW to add future pages just resort to (newapp) urls.py and add it in path:
    exemple:

    from django.urls import path
    from . import views

    urlpatterns = [
    path(' ', views.home, name='home'),
    path('about/', views.about, name='about'),
    path('newpage/', views.newpage,
    name='newpage'),
    ]


    THEN go to views.py in (newapp) and add view:

    #HOME

    def home(request):
    return render(request, "home.html", {})

    #ABOUT

    def about(request):
    return render(request, "about.html", {})

    #NEWPAGE

    def newpage(request):
    return render(request, "newpage.html", {})


    THEN go to templates and create the 'newpage.html'

    and So Forth ...
